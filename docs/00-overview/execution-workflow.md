# Execution Workflow

## Principle

Plan first, execute second, verify third.

## Standard Flow

1. Scope and success criteria are frozen.
2. Input and output contracts are documented.
3. Implementation is split by phase.
4. Testing and failure paths are validated.
5. Deployment and demo script are finalized.

## Documentation Rules

- Every significant change must map to one markdown file in docs.
- Keep each file outcome-oriented with explicit done criteria.
- Record assumptions, risks, and pending items.

## Status Update Template

Use this format in phase files:

- Status: Not Started | In Progress | Completed
